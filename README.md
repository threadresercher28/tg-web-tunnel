# tg-web-tunnel

Локальный HTTP(S)-прокси на Python, который позволяет ходить в сеть, когда прямой доступ ограничен.

- **HTTPS-трафик** (`CONNECT`, включая Telegram Desktop, браузеры и т.д.) идёт напрямую **TCP-туннелем** к целевому хосту — без промежуточного PHP-релея.
- **HTTP-трафик** (`GET/POST/PUT/DELETE/HEAD/OPTIONS/PATCH`) заворачивается через PHP-скрипт-релей (`prx.php`), который вы размещаете на своём шаред-хостинге. Прокси сам расжимает ответ (gzip/deflate/br) и корректно проксирует его клиенту.

Многопоточный (`ThreadingMixIn`), устойчив к обрывам соединений и падению отдельных запросов — один упавший клиент не кладёт весь сервер.

## Как это работает

```
Клиент (браузер / Telegram Desktop и т.д.)
        │
        ▼
   tghelp.py (локальный прокси, порт 8021)
        │
        ├── HTTPS (CONNECT) ──► напрямую TCP-туннель ──► целевой сервер
        │
        └── HTTP (GET/POST/...) ──► PHP-релей на хостинге (prx.php) ──► целевой сервер
```

## Требования

- Python 3.8+
- Зависимости из `requirements.txt`:
  ```
  requests==2.31.0
  urllib3==2.1.0
  ```
- PHP-скрипт-релей (`prx.php`), развёрнутый на любом доступном шаред-хостинге, для проксирования HTTP-запросов.

## Установка

```bash
git clone https://github.com/threadresercher28/tg-web-tunnel.git
cd tg-web-tunnel
pip install -r requirements.txt
```

## Настройка

Адрес PHP-релея задаётся константой `PHP_PROXY_URL` в начале `tghelp.py`:

```python
PHP_PROXY_URL = "https://your-domain.example/prx.php"
```

Пропишите туда адрес своего развёрнутого `prx.php`.

Прочие параметры (порт по умолчанию, таймауты, логирование) можно посмотреть/поменять в `config.py`.

## Запуск

```bash
python3 tghelp.py [порт]
```

Если порт не указан, используется `8021`.

```bash
python3 tghelp.py 8021
```

После запуска в консоли появится:

```
✅ Прокси запущен на 0.0.0.0:8021
   HTTP  -> через PHP-релей: https://your-domain.example/prx.php
   HTTPS -> прямой CONNECT-туннель (PHP не используется)
```

## Использование

Укажите в настройках приложения/системы/браузера HTTP-прокси:

```
Хост: 127.0.0.1 (или IP машины, где запущен tghelp.py)
Порт: 8021 (или тот, что указали при запуске)
```

Прокси поддерживает HTTP CONNECT, поэтому подходит и для HTTPS-сайтов, и для приложений, использующих системный HTTP-прокси (например, Telegram Desktop через *Settings → Advanced → Connection type → Use custom proxy*, тип **HTTP/Sockety** → HTTP-прокси).

## Ограничения

- HTTP-трафик зависит от доступности и лимитов вашего шаред-хостинга (PHP-скрипт `prx.php` должен быть развёрнут отдельно, в этот репозиторий он не входит).
- Для HTTPS PHP-релей не используется вовсе — используется прямой TCP CONNECT, поэтому если исходящие TCP-соединения на 443 порт заблокированы на уровне сети, HTTPS-туннель работать не будет.
